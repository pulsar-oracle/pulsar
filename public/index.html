How It Works

Three steps from real-world data to your Soroban contract.

1

Feeder Nodes Fetch

Independent operators run Pulsar feeder nodes that pull price data from multiple CEXs and APIs every 60 seconds.

2

On-Chain Aggregation

The Pulsar aggregator contract collects submissions, computes the median, and stores the result — all on Stellar Mainnet.

3

Your Contract Reads

Any Soroban contract reads a Pulsar feed with a single cross-contract call. No whitelisting, no API keys.

Consume a Feed

One cross-contract call is all it takes.

// Read the XLM/USD price inside your Soroban contract
pub fn get_xlm_price(env: Env, pulsar: Address) -> i128 {
    let client = pulsar_contracts::Client::new(&env, &pulsar);
    let feed = client.get(&symbol_short!("XLM_USD"))
        .expect("feed not found");
    feed.value // scaled by 1_000_000 → $1.05 = 1_050_000
}
